# Debug Changelog — Mobile Hero Overflow Fix

## Root Causes
1. **Tailwind v4 class conflict**: `cn()` is simple string concatenation (not tailwind-merge), so `px-0` doesn't override Container's `px-5` in Tailwind v4 where stylesheet order determines winner, not class attribute order.
2. **Gallery `w-[88vw]` overflow**: Viewport-relative widths on gallery figures caused content wider than the container, leading to page-level horizontal overflow.
3. **Missing `w-full min-w-0`**: The hero-copy div had `max-w-[22rem]` but no explicit width constraint, so in RTL grid layout it could grow wider than its grid column.

## Changes Made

### 1. Fixed Container padding override with `!important` modifier
- **File:** `src/components/pages/HomePage.tsx`
- **Change:** Changed `max-w-none px-0 sm:px-0 lg:max-w-[1160px] lg:px-8` to `max-w-none !px-0 sm:!px-0 lg:max-w-[1160px] lg:!px-8` on the Container className
- **Why:** In Tailwind v4, `px-0` doesn't override `px-5` from Container's own classes because `cn()` just concatenates strings. The `!` modifier ensures the override wins.
- **Revert:** Remove `!` prefixes from `px-0`, `sm:px-0`, and `lg:px-8`

### 2. Changed hero section overflow from `overflow-hidden` to `overflow-x-clip`
- **File:** `src/components/pages/HomePage.tsx`
- **Change:** Changed `className="relative overflow-hidden"` to `className="relative overflow-x-clip overflow-y-visible"` on the `<section id="home">` element
- **Why:** `overflow-x-clip` prevents horizontal overflow from escaping the hero section without affecting vertical layout (animations, sticky elements).
- **Revert:** Change back to `overflow-hidden`

### 3. Added `w-full min-w-0` to hero-copy div
- **File:** `src/components/pages/HomePage.tsx`
- **Change:** Added `w-full min-w-0` classes to the hero-copy div
- **Why:** Without explicit width, the grid item could grow to its `max-w-[22rem]` (352px) even when the grid column was narrower on small viewports, causing overflow in RTL.
- **Revert:** Remove `w-full min-w-0` from the hero-copy div class list

### 4. Gallery breakout with negative margins
- **File:** `src/components/pages/HomePage.tsx`
- **Change:** Added `-mx-5 sm:-mx-8` to gallery wrapper and `px-5 sm:px-8` to scroll container and label
- **Why:** Allows the gallery scroll container to extend edge-to-edge while text remains padded, preventing figures from overflowing the page.
- **Revert:** Remove `-mx-5 sm:-mx-8` from wrapper, remove `px-5 sm:px-8` from scroll container and label

### 5. Changed gallery figure widths from `vw` to `calc()`
- **File:** `src/components/pages/HomePage.tsx`
- **Change:** Changed first figure from `w-[88vw]` to `w-[calc(100vw-3rem)]`, others from `w-[76vw]` to `w-[calc(100vw-5rem)]`
- **Why:** `calc()` accounts for container padding, preventing figures from being wider than available space.
- **Revert:** Change back to `w-[88vw]` and `w-[76vw]` respectively

## Revert Status
- These are production fixes, not temporary debugging changes. No revert needed.
